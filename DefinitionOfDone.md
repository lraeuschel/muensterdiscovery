## Definition of Done (DoD)

A task / user story is **DONE** when all of the following criteria are met:

- [ ] **Acceptance criteria are fulfilled**  
  - All described requirements and use cases work as specified in the corresponding Issue.

- [ ] **No critical or obvious bugs**  
  - There are no known issues that significantly impact the usage of this feature.

- [ ] **Code is committed and builds successfully**  
  - All changes are pushed to the shared repository.  
  - The application builds/starts without errors (via npm run deploy).

- [ ] **Code review completed**  
  - At least one other team member has reviewed the code/ site and approved it.

- [ ] **Documentation is updated where needed**  
  - Relevant documentation (e.g. README) is updated.
